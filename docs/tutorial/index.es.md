# Aprende CML

Context Mapper Language (CML) es un DSL creado por el proyecto [Context Mapper](https://contextmapper.org/) para modelar Domain-Driven Design (DDD) de punta a punta, desde la visión estratégica hasta la implementación táctica.

## Introducción {#intro}

Esta guía te llevará a través de los componentes principales de CML. La referencia completa está disponible en la [Language Reference](https://contextmapper.org/docs/language-reference/) oficial.

### Estructura del Lenguaje

CML te permite describir tres niveles de modelado:

1.  **DDD Estratégico**: Context Maps, Dominios, Subdominios y Bounded Contexts.
2.  **Análisis de Requerimientos**: Casos de Uso, Historias de Usuario y Stakeholders.
3.  **DDD Táctico**: Agregados, Entidades, Servicios, Eventos, etc.
4.  **Capa de Aplicación**: Comandos, Flujos y Coordinación.
